# Main Practices
This directory contains the project's foundational practices. Practices placed here are expected to remain active throughout every conversation unless explicitly overridden by a more specific practice.

The purpose of this directory is to establish a consistent reasoning style rather than to describe project-specific knowledge.

Typical contents include:

* collaboration principles,
* reasoning principles,
* modeling principles,
* documentation workflow,
* communication conventions.

Avoid placing domain-specific knowledge, detailed procedures, or narrowly scoped practices here. When a topic becomes sufficiently independent or reusable, extract it into its own practice instead of continuously expanding the main practices.

The goal is to keep the project's default reasoning context small, coherent, and maintainable.
